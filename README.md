# Telegram File Gatekeeper Bot

A Telegram bot built with **Node.js** and the `node-telegram-bot-api v2` library, designed to share files through automatically generated links. Ideal for keeping files organized and secure in a private Telegram group without exposing it directly to end users.

---

## ✨ Key Features

- **🗄️ Telegram Group as database**: Files are stored in your Telegram group (it is recommended to keep the group private so that only admins send content to the bot, preventing unnecessary link generation).
- **🔗 Permanent Links**: Each file uploaded to the Telegram group receives a unique and permanent link.
- **🌐 Universal Access**: Links can be shared with anyone, even if they are not members of the Telegram group.
- **🛡️ Enhanced Privacy**: Your Telegram group (where you store the files) will not interact with users; therefore, if any file has copyright issues, the bot will bear the consequences, not the entire group with all your files. (As this feature can encourage piracy, please use the bot wisely).
- **📊 Zero cost per file**: When a user requests a file, the bot merely forwards it, so it consumes practically no resources regardless of the file size.

---

## ⚠️ Responsible Use

This bot is intended to facilitate organized and private file sharing.
**It must not be used to distribute copyright-protected content without authorization.**
The developer is not responsible for misuse of this tool.

---

## 🚀 How It Works

1. **Create a Telegram group** (preferably private) to serve as the file storage.
2. **Add the bot as an administrator** to the group, granting read and write permissions.
3. **Send files to the group** - the bot will automatically generate a unique link per file.
4. **Share links** with anyone; when clicked, the bot will send the corresponding file to the user.

## ⚙️ Setup

### 1. Clone the Repository

```bash
git clone https://github.com/antoinepdev/telegram-file-gatekeeper.git
cd telegram-file-gatekeeper
```

### 2. Install Dependencies

```bash
bun install
```

### 3. Configure Environment Variables

Create a `.env` file in the project root with the following variables:

| Variable           | Description                                        | How to Obtain                                                                                            |
| ------------------ | -------------------------------------------------- | -------------------------------------------------------------------------------------------------------- |
| `BOT_TOKEN`        | Bot authentication token                           | Generated by [@BotFather](https://t.me/BotFather)                                                        |
| `PRIVATE_GROUP_ID` | ID of the group where files will be stored         | Obtain by adding [@RawDataBot](https://t.me/RawDataBot) to the group and copying the `"chat"."id"` value |
| `BOT_URL`          | Public bot link (e.g., `https://t.me/YourBotName`) | Provided by @BotFather when creating the bot                                                             |

#### Example `.env` file:

```env
BOT_TOKEN=717jfjfngncn:AAFpl5chxhdvwk4n1yKpznj91tmqZ3chfjfr
PRIVATE_GROUP_ID=-10024748423698
BOT_URL=https://t.me/peliss_bot
```

### 4. Start the Bot

```bash
bun prod
```

If configured correctly, you will see the message `"Running..."` in the console.
